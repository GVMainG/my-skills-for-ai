# notes-manager

Управление Obsidian vault: анализ тегов, audit заметок, поиск, дневник, генерация профиля.

## Триггеры

| Фраза пользователя | Команда |
|---|---|
| "анализ тегов", "проанализируй теги", "какие теги поставить" | `analyze-tags` |
| "audit заметок", "проверь заметки", "найди проблемы в vault" | `audit` |
| "найди заметки", "поищи по vault", "найди про ...", "найди заметки о ..." | `search` |
| "дневниковая запись", "сделай запись в дневник", "помоги с дневником" | `daily-journal` |
| "создай профиль", "обнови профиль", "сгенерируй CLAUDE.md" | `generate-profile` |

## Команды

### analyze-tags

```bash
notes-manager analyze-tags <filepath>
  --auto-apply    # Автоматически применить рекомендации
  --dry-run       # Показать изменения без применения
```

**Что делает:** Анализирует содержимое заметки, сопоставляет с иерархией тегов из `TAGS_HIERARCHICAL.md`, рекомендует теги на добавление/удаление.

---

### audit

```bash
notes-manager audit <folder_path>
  --check-format           # Проверить frontmatter и длину
  --check-placement        # Проверить соответствие папки/тегов
  --find-duplicates        # Найти дубли (Jaccard similarity)
  --check-links            # Найти битые [[ссылки]]
  --check-tags             # Теги не из иерархии
  --auto-fix               # Автоисправление тривиальных проблем
  --all                    # Все проверки (по умолчанию)
```

**Что делает:** Проходит по всем .md файлам в папке, собирает отчёт по 5 категориям проблем, предлагает автоисправление.

---

### search

```bash
notes-manager search "<запрос>"
  --max-results 10
  --include-backlinks
  --scope <папка>
  --tags <тег1,тег2>
```

**Что делает:** Трёхуровневый поиск — теги → ключевые слова → бэклинки. Приоритизация по количеству бэклинков, совпадению тегов, дате изменения.

---

### daily-journal

```bash
notes-manager daily-journal
  --questions 5
  --find-connections    # Искать связи с прошлыми записями
  --no-create           # Только показать вопросы
```

**Что делает:** Генерирует адаптивные вопросы на основе недавних заметок, собирает ответы, ищет тематические связи с прошлыми дневниковыми записями, создаёт .md файл.

---

### generate-profile

```bash
notes-manager generate-profile
  --update              # Обновить существующий профиль
  --output <путь>       # Кастомный путь (default: CLAUDE.md)
  --sections <список>   # Конкретные секции
```

**Что делает:** Deep semantic analysis всех заметок vault — извлечение технических навыков, построение психологического портрета (Big Five, когнитивный стиль, ценности, противоречия), граф знаний, кластеризация. Результат — файл CLAUDE.md.

---

## Конфигурация

Файл: `~/.notes-manager/config.yaml`

При первом запуске создаётся автоматически. Запрашивает путь к vault.

```bash
notes-manager init
```

## Установка

```bash
cd notes-manager/
pip install -e . --break-system-packages
notes-manager init
```
